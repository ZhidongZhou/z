int func(int x){

  if (x > 0){
    r = 1;
  }
  if (x < 0){
    assert(r == 0);
  }
}

Suppose we symbolically execute the program, from an initial state in which the variable “x” has symbolic value “X”. Select all formulas below (including unsatisfiable ones) that correspond to path conditions for execution paths through this program at the point that the “assert” statement is reached.

Group of answer choices

  • r = 0

Question 3

Fuzzing vs Symbolic Execution

Fuzzing and dynamic symbolic execution are both techniques for generating inputs for programs under test. They are often said to be complementary in nature, in that they have opposite properties. Considering them as input-generation methods, match each with its properties below.

Fuzzing

Computationally cheap but imprecise: inputs are cheap to generate but might not follow an unexplored execution path.

Dynamic Symbolic Execution

Computationally expensive but precise: input generation is relatively expensive but inputs are guaranteed to follow unexplored program paths.

Markov Models, Consider the following program:

Suppose we are using coverage-guided fuzzing to fuzz this program, which takes a 2-character (two-byte) string as its input. Suppose we use the initial seed input “go”, i.e. bytes[0] == ‘g’ and bytes[1] == ‘o’.

The program has 3 execution paths. Therefore the coverage guided fuzzer can be in one of three states (assuming we have already run the program with the seed input):

  1. One execution path has been found, namely the initial execution path in which neither if-condition evaluates to true. The only “interesting” input that the fuzzer has remembered is the initial seed: “go”.
  2. Two execution paths have been found, the initial one and one in which the first if-condition evaluates to true. The fuzzer has two interesting inputs it has remembered: “go” and “ho”.
  3. All three execution paths have been found. The fuzzer has three interesting inputs it has remembered: “go”, “ho” and “hi”.

At each iteration, the coverage guided fuzzer chooses an interesting input at random to mutate. It mutates that input by first choosing which byte at random to mutate (either the first byte or the second byte), and then mutates that byte by replacing it with a randomly generated byte (possibly identical to its original value). It then runs the program on the newly generated input. If a new execution path is found, that new input is remembered as interesting. Otherwise it is discarded.
