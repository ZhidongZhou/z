    163 邮件发送

    ZDZ
    14-05-2020
    Hits
    IT › Terminal
    MacOS


    开始连接：

    1. 打开 Terminal 终端

    ⌘ + SPACE


    2. 连接163邮箱

    telnet smtp.163.com 25


    3. 同服务器进行 Auth 验证

    helo smtp.163.com

    返回： 250 OK


    4. 登陆: auth login

    auth login

    返回的： 334 dXNlcm5hbWU6 : 就是问你要 加密 过的 username 用户名。

    4.1. 用户名:

    (例): 你的用户名是： helloworld ⇢通过⇢ BASE64译码 ⇢你将得到⇢ aGVsbG93b3JsZA==

    BASE64 在线译码： https://www.base64encode.org/

    返回的： 334 UGFzc3dvcmQ6， 就是问你要加密过的 Password 用户名。


    4.2. 密码:

    由于163.com 要求自行开启授权（如下图）

    Tips: 纠结于IMAP / POP3 的可以参考下图

    成功获取授权后，您将得到_(16位的)_ 客户端授权密码， 此后这个客户端授权密码将代替你原本的163登陆密码当你使用 imap / pop3 进行邮箱登陆。

    ⚠️ 你同样需要再次使用 BASE64译码 ⇢对⇢ 客户端授权密码 进行译码。


    开始发邮件

    1. 发件人邮箱

    from mail: youremail@163.com

    返回：250 Mail OK

    ⚠️ from mail， 空格必须要加。


    2. 收件人邮箱**

    rcpt to: <reciver@???.com>

    ⚠️ rcpt to, 空格必须要加。

    返回：250 Mail OK


    3. 编写邮件内容:

    3.1. 开始编辑

    data

    返回： 354 End data with .


    3.2. 邮件主题

    输入 subject: +加+ 两次回车

    subject: + ⏎ + ⏎

    ⚠️ 最后注意subject: 后面不要用(接): test 或 hello, 会被视为 垃圾邮件 (554 DT:SPM)


    3.3. 编辑正文

    ⚠️ 防止邮件被默认为： 垃圾邮件 (554 DT:SPM) 在输入： data 开始编辑正文之后，要外加：· · ·

    FROM: youremail@163.com TO: reciver@???.com CC: youremail@163.com

    开始输入正文 · · ·


    3.4. 发送邮件：

    . + ⏎

    (英文句号.) +接+ 回车 完成并发送邮件

    返回： 250 Mail Ok 表示成功

    ⚠️ 如果出现554 DT:SPM则被当作垃圾邮件了， 请往下滑对症下药。


    500 Error: bad syntax

    登陆失败 - 500 Error: bad syntax

    1. 检查 from mail: & rcptmail: 的格式， 主要空格
    2. 检查 hostname (不确定是否与500 Error 相关, 不过网上大量方法指向 hostname)

    2.1 hostname: 应该返回: 127.0.0.1

    1. MacOS: 如果不是 127.0.0.1 terminal 中 命令行输入: hostname 127.0.0.1 ， 进行改写。

    hostname 127.0.0.1

    1. Windows: 对比修改 /etc/hosts , 将开通为 127.0.0.1 改为：


    554 DT:SPM

    554 DT:SPM - 垃圾邮件

    554 DT:SPM 发送的邮件内容包含了未被许可的信息，或被系统识别为垃圾邮件。请检查是否有用户发送病毒或者垃圾邮件；

    错误原因：网易反垃圾邮件级别： * “低级”为：不做用户级过滤； * “中级”为：如果“发件人(From)”不在自己的[通讯录]和[白名单]里，并且“收件人(To)”和“抄送人(Cc)”不包括自己的地址，则认为是[垃圾邮件]； * “高级”为：如果邮件的“发件人(From)”地址不在自己的[通讯录]和[白名单]列表里，则认为来信是[垃圾邮件]。请谨慎选择此级别。

    554 DT:SPM - 解决：

    ⚠️ 在输入： DATA 之后 要外加： FROM: youremail@163.com & TO: reciver@???.com，CC: youremail@163.com 最后注意subject: 不要用: test 或 hello
