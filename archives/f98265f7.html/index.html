    在不使用 deepcopy() 的前提下， 实现对 { { 嵌套字典 } } 的深拷贝



    为什么要深拷贝?

    当添加对拷贝的dictionary进行， 添加、 删除、 等 ···, 修操作的时候， 原字典 与 拷贝的字典 之间会产生 相互影响 (一个改变另一个也会随之改变)。

    深拷贝 deepcopy:



    上述 相互影响 可通过 deepcopy() 解决， 由于深拷贝有别于正常逻辑下的普通拷贝和浅拷贝。


    深复制: 将原有数据 完整的再复制 , 产生一个 全新独立 的个体。



    浅拷贝: shallowcopy

    浅拷贝: 在原有数据上 打上一个标签， 当标签被数据被修改， 原有数据会被改变， 其他的标签也会被改变。

    那么举一个复杂字典的例子🌰: { 1:{ 2:'TWO' } }
