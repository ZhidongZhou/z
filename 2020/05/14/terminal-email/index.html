Email via Terminal — 163邮箱

相·遇
14-05-2020
hack

打开 Terminal 终端

连接163邮箱

telnet smtp.163.com 25

登陆前，需要用 helo 和 elho 向服务器进行验证

helo smtp.163.com

返回： 250 OK

elho smtp.163.com

返回：250 OK

250-mail
250-PIPELINING
250-AUTH LOGIN PLAIN 
250-AUTH=LOGIN PLAIN
250-coremail *********(略)***********
250-STARTTLS
250 8BITMIME

登陆: auth login

auth login

返回的： 334 dXNlcm5hbWU6， 就是问你要加密过的 username 用户名。

用户名:

(例)你的用户名是： helloworld ⇢ BASE64译码得到 ⇢ aGVsbG93b3JsZA==

可以使用 BASE64 在线译码： https://www.base64encode.org/

返回的： 334 UGFzc3dvcmQ6， 就是问你要加密过的 Password 用户名。

密码:

由于163.com 要求自行开启授权（如下图）， 授权后你的登陆密码也将由(16位的)客户端授权密码替换。

Tips: 纠结于IMAP / POP3 的可以参考下图

发件人邮箱

from mail: youremail@163.com

返回：250 Mail OK

收件人邮箱

rcpt to: <receiveremail@???.com>

返回：250 Mail OK

登陆失败 500 Error: bad syntax

1. 检查 from mail: <> & rcptmail: <> 的格式， 主要空格

2. 检查 hostname (不确定是否与500 Error 相关, 不过网上大量方法指向 hostname)

hostname

应该返回: 127.0.0.1

如果不是 127.0.0.1

Mac： 命令行输入: hostname 127.0.0.1 ， 进行改写。

hostname: 127.0.0.1

或 Windows 对比修改 /etc/hosts , 将开通为 127.0.0.1 改为：

127.0.0.1 localhost.localdomain localhost

编写邮件内容:

编写邮件 data

data

返回： 354 End data with .

**邮件主题 subject:**

subject: 主题

**编辑邮件内容：‘两次回车’**

输入： . (英文句号) + 回车 完成并发送邮件

返回： 250 Mail Ok 表示成功

554 DT:SPM

常见错误： 554 DT:SPM 163 smtp4,HNxpCgBX5yrd6LxemSLDBA–.113S6 1589439356,please see http://mail.163.com/help/help\_spam\_16.htm?ip=124.188.99.158&hostid=smtp4&time=1589439356

554 DT:SPM 发送的邮件内容包含了未被许可的信息，或被系统识别为垃圾邮件。请检查是否有用户发送病毒或者垃圾邮件；

错误原因：网易反垃圾邮件级别

  • “低级”为：不做用户级过滤；
  • “中级”为：如果“发件人(From)”不在自己的[通讯录]和[白名单]里，并且“收件人(To)”和“抄送人(Cc)”不包括自己的地址，则认为是[垃圾邮件]；
  • “高级”为：如果邮件的“发件人(From)”地址不在自己的[通讯录]和[白名单]列表里，则认为来信是[垃圾邮件]。请谨慎选择此级别。

解决： 554 DT:SPM

在输入： DATA 之后 要外加： FROM: youremail@163.com & TO: reciver@???.com，CC: youremail@163.com 最后注意subject: 不要用: test 或 hello
