在不使用 deepcopy() 的前提下， 实现对{ { 嵌套字典 } }的深拷贝


为什么要深拷贝?

当添加对拷贝的dictionary进行， 添加、 删除、 等 ···, 修操作的时候，原字典 与 拷贝的字典 之间会产生_ 相互影响_ (一个改变另一个也会随之改变)。

深拷贝 deepcopy:



上述 _相互影响_ 可通过 deepcopy() 解决， 由于深拷贝有别于正常逻辑下的普通拷贝和浅拷贝。


深复制: 将原有数据 完整的再复制 , 产生一个 全新独立 的个体。



浅拷贝: shallowcopy

浅拷贝: 在原有数据上 打上一个标签， 当标签被数据被修改， 原有数据会被改变， 其他的标签也会被改变。



深 v 浅

从上可以看出， 普通拷贝 d2 = d1 与 copy() 是有区别的， 普通拷贝在任何情况下都会被原始字典中的变化所改变， 而浅拷贝可以应对简单的字典数据， 然而不足以应对复杂的字典数据。


那么举一个复杂字典的例子🌰: { 1:{ 2:'TWO' } }

从上可以看出, 当原始字典存在多层嵌套时， 浅拷贝d2 = copy.copy(d1) 和 普通拷贝d2 = d1 均不能保证拷贝数据不受原始数据改变的影响； 从而在这种情况下我们需要使用copy.deepcopy()
